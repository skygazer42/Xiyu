# Web 前端业务用户操作手册

## 一、适用范围

本手册适用于使用 Xiyu Web 前端开展音频转写、会议记录整理、热词维护及实时听记工作的业务人员。

本手册重点说明以下内容：

- 系统访问方式
- 各页面业务用途
- 常用操作流程
- 结果查看与导出要求
- 常见问题处置建议

部署方式、接口调用、模型配置等技术事项，不在本手册说明范围内。

技术同事如需查看部署、接口映射、前后端联调及排障说明，请参见：

- `docs/WEB_UI_TECHNICAL.md`

## 二、访问方式

### （一）系统访问地址

系统推荐统一从以下地址访问：

```text
http://<服务器IP>:18200
```

常用访问入口如下：

- Web 页面：`http://<服务器IP>:18200`
- 接口文档：`http://<服务器IP>:18200/docs`
- 服务状态：`http://<服务器IP>:18200/health`

### （二）浏览器要求

建议使用以下浏览器访问系统：

- Chrome 最新版本
- Edge 最新版本

如浏览器版本过旧，可能出现页面显示异常、文件上传异常或麦克风权限调用失败等情况。

## 三、页面功能概览

系统左侧菜单通常包含以下页面：

- `转写`：用于文件转写、URL 转写、结果查看、结果导出及历史记录调用
- `实时转写`：用于现场讲话、会议速记等实时听记场景
- `热词管理`：用于维护专有名词、机构名称、人名、地名及纠错规则
- `配置管理`：用于系统参数查看与设置，普通业务使用场景一般不建议调整
- `系统监控`：用于查看服务在线状态、请求量、成功率及处理速度等运行信息

如仅涉及日常会议转写，建议重点使用以下页面：

- `转写`
- `实时转写`
- `热词管理`

## 四、业务使用场景对应关系

| 使用场景 | 推荐页面 | 推荐操作 |
|---|---|---|
| 本地录音转写 | `转写` | 上传文件后执行 `开始转写` |
| 网盘直链音频或视频转写 | `转写` | 切换至 `URL转写` 后提交链接 |
| 超长会议处理 | `转写` | 使用 `提交文件任务` 并在 `任务队列` 查看结果 |
| 嘈杂录音优化 | `转写` | 开启 `ClearVoice 降噪` |
| 多人会议分说话人整理 | `转写` | 开启 `说话人识别` |
| 正式成稿整理 | `转写` | 使用 `全量优化` |
| 现场边说边记 | `实时转写` | 连接麦克风后开始录制 |
| 专有名词识别纠偏 | `热词管理` | 维护热词或纠错内容 |

## 五、转写页面操作说明

`转写` 页面为系统主要业务页面，适用于录音文件、会议音频、访谈音频、讲座音频等内容的处理。

### （一）输入方式

页面顶部提供两种输入方式：

- `文件上传`
- `URL转写`

#### 1. 文件上传

适用于音频文件已保存至本机的场景。

常见支持格式包括：

- `wav`
- `mp3`
- `m4a`
- `flac`
- `ogg`
- 其他系统支持的常见音频格式

操作流程如下：

1. 进入 `转写` 页面。
2. 保持在 `文件上传` 模式。
3. 点击上传区域选择文件，或将文件直接拖入上传区域。
4. 在右侧设置转写选项。
5. 根据业务需要选择启动方式。

#### 2. URL 转写

适用于音频或视频文件已存放于可直接访问链接的场景。

使用要求如下：

- 链接协议须为 `http://` 或 `https://`
- 链接建议直接指向音频或视频文件本身
- 需要登录、鉴权或临时口令的链接，可能无法正常处理

操作流程如下：

1. 切换至 `URL转写`。
2. 填写文件直链。
3. 在右侧设置转写选项。
4. 点击 `开始转写`。
5. 提交后在下方 `任务队列` 查看处理进度及结果。

### （二）常用选项说明

#### 1. 服务地址

默认建议保持为：

- `当前服务 (相对路径 · 推荐)`

普通业务使用场景一般不建议改为自定义地址，除非技术人员已明确说明。

#### 2. 目标模型（Router）

如系统显示 `目标模型（Router）` 下拉框，业务使用场景建议保持为：

- `自动 (Router 策略)`

仅在开展模型比对或技术人员明确要求时，才切换为 `Qwen3 / PyTorch / ONNX / SenseVoice / Whisper` 等指定模型。

#### 3. ClearVoice 降噪

建议在以下场景开启：

- 录音存在明显底噪
- 会场环境复杂
- 麦克风距离较远
- 音频清晰度不足

一般建议如下：

- 原始音频质量较高：可不开启
- 原始音频环境嘈杂：建议开启

开启后，页面中的 `下载降噪音频` 按钮可用于另存降噪后的音频文件。

#### 4. 说话人识别

适用于多人会议、访谈、讨论、座谈等场景。

开启后，结果中通常会体现：

- 不同发言人的分段
- 带发言人的时间轴
- 更适合会议纪要整理的结构化内容

如仅为单人发言场景，通常无需开启。

#### 5. 临时热词

适用于单次转写过程中临时补充专有名词。

适用内容包括：

- 单位名称
- 项目名称
- 人名
- 地名
- 生僻缩写

填写方式：

- 多个词之间使用空格分隔

功能特点：

- 仅对本次转写生效
- 不会自动保存至热词库

#### 6. 高级 asr_options

该项为高级功能，普通业务使用场景一般无需填写。

如不明确该项用途，建议保持空白。

### （三）三类主要启动方式

#### 1. 开始转写

适用情形：

- 普通音频
- 中短录音
- 需要较快查看结果

操作流程：

1. 上传 1 个或多个文件。
2. 设置相关选项。
3. 点击 `开始转写`。

功能特点：

- 单文件时，结果直接在当前页面显示
- 多文件时，系统按批量方式处理，并优先展示一个成功结果

#### 2. 提交文件任务

适用情形：

- 超长会议录音
- 大文件处理
- 不宜长期等待页面同步返回

页面提示中已明确建议：

- `3-4 小时会议建议使用队列`

操作流程：

1. 上传文件。
2. 设置相关选项。
3. 点击 `提交文件任务`。
4. 在下方 `任务队列` 查看状态。

常见任务状态包括：

- `等待中`
- `处理中`
- `完成`
- `失败`

任务完成后，可直接使用以下功能：

- `查看`
- 将任务结果加载至主结果区继续整理

#### 3. 全量优化 / 全量对比

该组按钮适用于对成稿质量要求较高的场景。

`全量对比`：

- 并发调用多个模型
- 用于比较不同模型结果

`全量优化`：

- 并发调用多个模型
- 再结合 LLM 进行融合润色
- 更适用于正式会议稿、上报材料、纪要整理等场景

功能特点：

- 耗时通常高于普通转写
- 更适用于重要材料
- 推荐在单文件场景下使用

### （四）转写结果查看

转写完成后，页面右侧或下方通常显示以下内容：

- 完整文本
- 带时间轴的句子列表
- 发言人分段
- 历史记录

如已开启说话人识别，结果更适合用于会议纪要整理与分角色校对。

### （五）结果导出

结果区 `下载` 按钮支持以下格式：

- `纯文本 (.txt)`
- `Markdown (.md)`
- `Word (.doc)`
- `字幕文件 (.srt)`
- `WebVTT (.vtt)`
- `JSON 数据 (.json)`

业务场景中常用格式通常包括：

- `.txt`：便于继续粘贴至办公文档整理
- `.doc`：便于直接在 Word 中打开和编辑
- `.srt`：适用于字幕整理场景

### （六）历史记录

页面下方设有 `转写历史`。

主要功能包括：

- 自动保存已完成的转写结果
- 支持关键词搜索
- 支持重新查看某次结果
- 支持从历史记录直接导出
- 支持删除单条记录或清空全部记录

该功能适用于反复查看已处理会议材料的场景。

## 六、实时转写页面操作说明

`实时转写` 页面适用于现场讲话、会议速记、临时发言记录等场景。

### （一）主要功能

- 建立 WebSocket 实时通道
- 调用本机麦克风
- 边录音边显示文字

### （二）操作流程

1. 进入 `实时转写` 页面。
2. 确保浏览器已授予麦克风权限。
3. 选择识别模式。
4. 点击开始录制。
5. 发言结束后点击停止录制。

### （三）模式说明

- `2pass`：同时给出在线结果和更稳定的最终结果，推荐使用
- `online`：更强调实时性
- `offline`：更强调完整段落结果

### （四）使用建议

- 日常业务记录建议优先使用 `2pass`
- 发言结束后建议等待最终结果刷新，再复制正文

页面还支持以下辅助操作：

- 复制文本
- 清空内容
- 查看连接状态

## 七、热词管理页面操作说明

`热词管理` 页面用于维护系统常见专有词，提高识别准确性与文本规范性。

页面主要分为以下四类内容：

- `强制热词`
- `上下文热词`
- `规则`
- `纠错`

### （一）强制热词

适用范围：

- 单位名称
- 平台名称
- 项目名称
- 固定术语

录入方式：

- 每行填写一个词

常见操作：

- `更新全部`
- `追加热词`
- `从文件重载`

### （二）上下文热词

适用于较长会议、专题讨论、政策类材料等场景。

与强制热词相比，上下文热词更适合作为提示性词汇使用，常见内容包括：

- 专题词汇
- 背景术语
- 会议上下文提示内容

### （三）规则

适用于固定格式替换、术语规范化等场景。

如不明确规则写法，建议暂不修改，并交由管理员或技术人员维护。

### （四）纠错

适用于维护“常错词 -> 正确词”的对应关系。

常见内容包括：

- 错写人名
- 高频错别字
- 固定简称误识别

## 八、配置管理页面使用原则

`配置管理` 页面更适用于管理员或熟悉系统参数的人员。

普通业务使用场景可按以下原则理解：

- 可查看
- 不建议随意修改

如确需使用，建议仅调整明确理解其作用的开关，例如：

- 热词相关设置
- LLM 启用状态
- 部分文本后处理选项

以下情况建议不自行修改：

- 不明确参数作用
- 修改后可能影响全体使用人员
- 未得到技术人员说明

## 九、系统监控页面使用说明

`系统监控` 页面主要用于判断系统是否正常运行。

建议重点关注以下指标：

- 服务是否在线
- 总请求数
- 成功率
- 平均处理速度

如发现以下情况：

- 服务离线
- 失败率明显上升
- 长时间无响应

建议先保留页面截图，再联系技术人员排查。

## 十、推荐使用规范

### （一）普通会议录音

建议配置如下：

- 页面：`转写`
- 方式：`文件上传`
- 开关：`说话人识别 = 开`
- 如环境嘈杂：`ClearVoice 降噪 = 开`
- 输出：转写完成后导出 `.doc` 或 `.txt`

### （二）超长会议

建议配置如下：

- 页面：`转写`
- 方式：上传文件后使用 `提交文件任务`
- 处理完成后在 `任务队列` 查看结果

### （三）正式材料整理

建议配置如下：

- 页面：`转写`
- 方式：单文件
- 开关：`说话人识别 = 开`
- 如对成稿要求较高：使用 `全量优化`

### （四）嘈杂录音

建议配置如下：

- 优先开启 `ClearVoice 降噪`
- 如有需要，可额外下载降噪音频进行核对

### （五）人名、机构名易识别错误

建议处理方式如下：

- 临时场景：使用 `临时热词`
- 长期场景：在 `热词管理` 中维护

## 十一、常见问题

### （一）页面无法打开

建议优先检查以下事项：

- 地址是否为 `http://<服务器IP>:18200`
- 当前网络是否能够访问服务器
- 技术人员是否已启动服务

### （二）上传后长时间无结果

建议优先判断以下情况：

- 文件较长：建议改用 `提交文件任务`
- 网络波动：建议刷新页面后重新查看任务队列
- 服务繁忙：建议稍后再试

### （三）URL 转写失败

常见原因包括：

- URL 不是直链
- 链接需要登录
- 链接已过期
- 链接并非音频或视频文件

### （四）说话人区分不准确

该情况在以下场景较为常见：

- 说话人频繁打断
- 麦克风距离较远
- 会场回声较重
- 多人同时发言

建议处理方式如下：

- 尽量使用更清晰的原始录音
- 必要时配合 `ClearVoice 降噪`
- 最终结果仍建议人工校对

### （五）导出 Word 后格式不理想

建议优先尝试以下方式：

- 导出 `.txt`
- 或导出 `.md`

随后再在 Word 中完成排版整理。

### （六）配置修改后效果异常

如无法确认已修改内容，建议：

- 暂停继续修改
- 联系技术人员协助恢复

普通业务使用建议尽量集中在以下页面完成：

- `转写`
- `实时转写`
- `热词管理`

## 十二、业务办理建议

如无特殊要求，建议按以下默认方式开展操作：

- 文件转写：上传文件后点击 `开始转写`
- 多人会议：开启 `说话人识别`
- 嘈杂录音：开启 `ClearVoice 降噪`
- 超长会议：使用 `提交文件任务`
- 正式材料：使用 `全量优化`
- 常错专有名词：纳入 `热词管理`

如系统可正常访问，但无法确定参数选择方式，建议优先保持默认值，不随意修改 `配置管理` 页面中的系统参数。
